# Автоматизация создания минимального контейнера

## Цель
Разработать скрипт-обертку, автоматизирующий процесс создания изолированного окружения,аналогичный базовой функциональности движков контейнеризации (например, Docker или containerd).

## Задание
1. Напишите скрипт на языке Bash (mycontainer.sh), который принимает команду для выполнения в качестве аргумента.
2. Скрипт должен выполнять следующие действия последовательно:
    * Создавать уникальную контрольную группу для данного запуска.
    * Устанавливать ограничение на количество процессорного времени (CPU quota) или памяти.
    * Запускать указанную команду в новых пространствах имен (PID, UTS, NET, MNT).
    * Обеспечивать корректную очистку ресурсов (удаление контрольной группы) после завершения работы команды, даже в случае ошибки.
3. Протестируйте скрипт, запустив внутри него интерактивную оболочку и проверив
   ограничения

## Технические требования
* Использование конструкции trap для обработки сигналов и очистки ресурсов.
* Проверка кодов возврата запускаемых процессов.
* Скрипт должен быть безопасным: не оставлять «висящих» cgroup или процессов после
  завершения.

## Критерии оценки
* Скрипт принимает аргументы командной строки.
* Реализована изоляция минимум по 4 типам namespaces (PID, UTS, NET, MNT).
* Реализовано ограничение ресурсов (CPU или Memory).
* Реализован механизм очистки (cleanup) ресурсов после завершения работы.
* Код скрипта содержит комментарии и обработку ошибок.
